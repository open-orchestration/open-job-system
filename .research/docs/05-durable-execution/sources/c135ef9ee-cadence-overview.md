# Introduction

Cadence is a new developer friendly way to develop distributed applications.

It borrows the core terminology from the workflow-automation space. So its concepts include [workflows](https://cadenceworkflow.io/docs/concepts/workflows) and [activities](https://cadenceworkflow.io/docs/concepts/activities). Workflows can react to [events](https://cadenceworkflow.io/docs/concepts/events) and return internal state through [queries](https://cadenceworkflow.io/docs/concepts/queries).

The [deployment topology](https://cadenceworkflow.io/docs/concepts/topology) explains how all these concepts are mapped to deployable software components.

The [HTTP API reference](https://cadenceworkflow.io/docs/concepts/http-api) describes how to use HTTP API to interact with Cadence server.
