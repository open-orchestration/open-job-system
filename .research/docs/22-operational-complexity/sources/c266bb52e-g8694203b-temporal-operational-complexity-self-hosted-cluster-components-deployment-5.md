Simplifies the deployment and management of self-hosted Temporal clusters across Kubernetes, EKS, and local environments.
AboutSKILL.mdFAQ
The Temporal Cluster Deployment skill provides comprehensive guidance and standardized configurations for setting up Temporal orchestration engines. It covers everything from local development using Docker Compose to production-grade Kubernetes deployments using Helm, including specific optimizations for Amazon EKS and PostgreSQL integrations. This skill is essential for developers and DevOps engineers who need to initialize database schemas, manage Helm values, and verify cluster health for scalable workflow orchestration.
## Key Features
01Local development setup using Docker Compose and SQLite
02Automated Helm chart configurations for Kubernetes and EKS
03Step-by-step database schema initialization and migration guides
04EKS-specific ingress and IAM role configuration templates
050 GitHub stars
06Production-ready PostgreSQL and Elasticsearch integration patterns
## Use Cases
01Configuring Amazon EKS with RDS PostgreSQL and ALB ingress for Temporal
02Setting up a local Temporal development environment for workflow testing
03Deploying a high-availability Temporal cluster to production Kubernetes
