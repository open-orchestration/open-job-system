# Azure Workflow and Data-Integration Services: Logic Apps and Data Factory

A landscape survey of two Azure services relevant to a job/work-execution system: Azure Logic Apps for workflow automation and integration, and Azure Data Factory for data integration and ETL/ELT. Each section draws strictly from that service's own documentation.

## Azure Logic Apps

Azure Logic Apps is a cloud platform where you can create and run automated workflows in, across, and outside the software ecosystems in an enterprise or organization, greatly reducing or removing the need to write code when workflows must connect and work with resources from different components such as services, systems, apps, and data sources [c41adc238]. It includes low-code-no-code tools for working with legacy, modern, and cutting-edge systems that exist in the cloud, on premises, or in hybrid environments, for example a visual designer along with prebuilt operations that act as building blocks by giving access to resources and completing common tasks such as getting, sending, and managing data [c41adc238].

Per the service's core terminology, a workflow is "a series of operations that define a task, business process, or workload," and each workflow "always starts with a single trigger operation, after which you must add one or more action operations" [c41adc238]. A trigger is the first operation in any workflow that specifies the criteria to meet before running any subsequent operations, such as getting an email or detecting a new file in a storage account, and an action is each subsequent operation that follows the trigger in the workflow [c41adc238]. A logic app is the Azure resource you create to build a workflow, available as a Consumption resource that supports a single workflow hosted in multitenant Azure Logic Apps or a Standard resource that supports multiple workflows hosted in single-tenant Azure Logic Apps, App Service Environment (ASE) v3, or a partially connected environment [c41adc238].

Logic Apps connects to resources through connectors: built-in connector operations run natively on the Azure Logic Apps runtime, while managed connector operations are Microsoft-published proxies for a service or system's REST API that are hosted and run in global, multitenant Azure [c41adc238]. The platform provides 1,400+ prebuilt connectors spanning Azure services, Office 365, database servers such as SQL and Oracle, enterprise systems such as SAP and IBM MQ, and file shares such as FTP and SFTP [c41adc238]. For control flow, a workflow can use conditions and switches to determine the next action — for example comparing each order to a cost threshold and routing to manual review when the condition is true [c41adc238].

Although building workflows usually requires no code, Logic Apps supports running custom code: in multitenant Azure Logic Apps you can write and run JavaScript code snippets directly within a workflow, and for workflows that run in single-tenant Azure Logic Apps, App Service Environment (ASE) v3, or partially connected environments, you can write and run JavaScript code snippets, .NET code, C# scripts, and PowerShell scripts directly within the workflow [c41adc238]. For more complex, structured code you can create and call functions using the Azure Functions platform [c41adc238]. A logic app workflow always starts with a single trigger that fires when a condition is met — many triggers include scheduling capabilities that control how often the workflow runs — after which one or more actions run operations that process, handle, or convert data or advance the workflow to the next step [c41adc238].

## Azure Data Factory

Azure Data Factory is a managed cloud service built for complex hybrid extract-transform-load (ETL), extract-load-transform (ELT), and data integration projects, providing a service that can orchestrate and operationalize processes to refine enormous stores of raw data into actionable business insights [cb73303e2]. The documentation defines it as "the cloud-based ETL and data integration service that allows you to create data-driven workflows for orchestrating data movement and transforming data at scale" [cb73303e2]. Using Azure Data Factory, you can create and schedule data-driven workflows (called pipelines) that can ingest data from disparate data stores, and build complex ETL processes that transform data visually with data flows or by using compute services such as Azure HDInsight Hadoop, Azure Databricks, and Azure SQL Database [cb73303e2].

Data Factory is composed of key components that work together to compose data-driven workflows with steps to move and transform data: Pipelines, Activities, Datasets, Linked services, Data Flows, and Integration Runtimes [cb73303e2]. A pipeline is a logical grouping of activities that performs a unit of work, and the activities in a pipeline can be chained together to operate sequentially or operate independently in parallel [cb73303e2]. Activities represent a processing step in a pipeline, and Data Factory supports three types: data movement activities, data transformation activities, and control activities [cb73303e2]. Datasets represent data structures within the data stores, and linked services define the connection information needed for Data Factory to connect to external resources [cb73303e2].

For connectivity, Azure Data Factory provides broad connectivity support for connecting to different data sources, useful when you want to pull or write data from different data sources [cb73303e2]. Triggers represent the unit of processing that determines when a pipeline execution needs to be kicked off, with different types of triggers for different types of events, and Azure Data Factory allows you to automate data processing using custom event triggers that automatically execute an action when a certain event occurs [cb73303e2]. A pipeline run is an instance of pipeline execution, typically instantiated by passing arguments to the parameters defined in pipelines, where arguments can be passed manually or within the trigger definition [cb73303e2]. For transformation, ADF mapping data flows let data engineers build and maintain data transformation graphs that execute on Spark without needing to understand Spark clusters or Spark programming, and ADF also supports external activities for executing transformations on compute services such as HDInsight Hadoop, Spark, Data Lake Analytics, and Machine Learning [cb73303e2]. Control flow is an orchestration of pipeline activities that includes chaining activities in a sequence, branching, defining parameters at the pipeline level, and passing arguments while invoking the pipeline on-demand or from a trigger, and also includes custom-state passing and looping containers such as For-each iterators [cb73303e2]. For security, Azure Data Factory offers integrated security features such as Entra ID integration and role-based access control to control access to dataflows [cb73303e2].
