# Decision log

The ADRs in this directory constitute the project's **decision log** — one architectural
decision plus its rationale per file. Records use the lean MADR-lite shape: YAML
frontmatter (`status`, `date`) over Context and Problem Statement → Decision →
Consequences. Files are numbered `NNNN-kebab-title.md`; `status` is one of `accepted`,
`rejected`, `deprecated`, or `superseded by ADR-NNNN`.

The consolidated MADR 4.0 records produced before this layout live in
[`../DECISIONS.md`](../DECISIONS.md) and are authoritative until each is split into its own
file here (see [0001](0001-adopt-adr-decision-log.md)).
