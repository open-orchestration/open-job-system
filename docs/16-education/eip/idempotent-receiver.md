Even when a sender application only sends a message once, the receiver application may receive the message more than once.

How can a message receiver deal with duplicate messages?

![](https://www.enterpriseintegrationpatterns.com/img/)

Design a receiver to be an _Idempotent Receiver_--one that can safely receive the same message multiple times.

The term _idempotent_ is used in mathematics to describe a function that produces the same result if it is applied to itself, i.e. f(x) = f(f(x)). In [_Messaging_](https://www.enterpriseintegrationpatterns.com/patterns/messaging/Messaging.html) this concepts translates into a message that has the same effect whether it is received once or multiple times. This means that a message can safely be resent without causing any problems even if the receiver receives duplicates of the same message.

Idempotency can be achieved through two primary means:

1. Explicit "de-duping", i.e. the removal of duplicate messages.
2. Defining the message semantics to support idempotency.
